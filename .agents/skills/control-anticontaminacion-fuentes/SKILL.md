# Propósito

Usar esta skill para evitar que el plan de empresa se construya con información incorrecta, obsoleta, ajena, no verificada o mal interpretada.

La función principal es proteger la honestidad documental del proyecto.

# Cuándo usar esta skill

Usar antes de:

* usar una fuente en una matriz;
* pasar una fuente a un consolidado;
* convertir una hipótesis en afirmación;
* citar benchmarks;
* usar documentos legales;
* usar ayudas o convocatorias;
* recuperar documentos históricos;
* reutilizar contenidos de otros planes de empresa;
* aceptar análisis generados por IA como evidencia.

# Entradas esperadas

* Fuente o fragmento a evaluar.
* Origen de la fuente.
* Fecha o vigencia estimada.
* Uso previsto dentro del plan.
* Carpeta o sección destino.
* Relación con otras fuentes.
* Nivel de verificación disponible.

# Salidas esperadas

Un dictamen de control con:

* tipo de riesgo;
* nivel de riesgo: bajo, medio, alto, crítico;
* uso permitido;
* uso prohibido;
* verificación requerida;
* decisión recomendada;
* observaciones.

# Procedimiento

1. Identificar el origen de la fuente.
2. Determinar si pertenece al proyecto vigente o a un antecedente histórico.
3. Revisar si el contenido corresponde al contexto Zaragoza / modelo urbano actual.
4. Separar dato, interpretación, hipótesis y recomendación.
5. Detectar si hay datos sin trazabilidad.
6. Detectar si la fuente es legal, financiera, de ayudas, ubicación, competencia o mercado.
7. Asignar riesgo de contaminación.
8. Definir uso permitido.
9. Definir uso prohibido.
10. Marcar verificación externa obligatoria cuando aplique.
11. Emitir dictamen.

# Reglas obligatorias

* No usar SISTREG como contenido del Taller.
* No usar documentos rurales como base del modelo vigente.
* No usar benchmarks como cumplimiento legal.
* No usar fuentes antiguas como vigentes sin comprobación.
* No usar ayudas 2021/2022 como oportunidad actual.
* No usar datos de precios sin fecha y fuente.
* No usar afirmaciones absolutas sobre mercado sin verificación.
* No tratar análisis IA como fuente primaria.
* No convertir hipótesis en hechos.

# Límites / No hacer

No hacer:

* corregir el plan redactando texto nuevo;
* decidir viabilidad de negocio por sí sola;
* sustituir asesoría legal, técnica, fiscal o PRL;
* mover archivos;
* importar fuentes;
* cerrar decisiones humanas.

# Criterio de utilidad para Marketing / Producto

La skill debe ayudar a Marketing a:

* evitar afirmaciones débiles;
* proteger la credibilidad del plan;
* separar oportunidad real de optimismo;
* identificar qué se puede defender ante evaluadores;
* decidir qué necesita investigación adicional;
* preservar la coherencia entre propuesta de valor, mercado, legal, operación y viabilidad.

# Criterio de cierre

La skill se considera completada cuando cada fuente o bloque evaluado queda con un uso permitido claro y una advertencia de riesgo si aplica.

# Evidencia esperada

* Dictamen anticontaminación.
* Lista de riesgos.
* Lista de verificaciones externas requeridas.
* Decisiones humanas pendientes.
