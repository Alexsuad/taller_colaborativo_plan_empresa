# 03_Mapa de temas y subtemas del plan de empresa

## Fase 0.0 — Organización documental de marketing

**Proyecto:** Taller Colaborativo Artesanal en Zaragoza  
**Estado:** Mapa maestro de temas para extracción documental  
**Uso:** Guía para buscar información en las 31 fuentes disponibles en NotebookLM / repositorio

---

## 1. Objetivo

Definir el listado amplio de temas y subtemas que deberán investigarse dentro de las fuentes disponibles antes de redactar el plan de empresa.

Este documento sirve para:

- ordenar la búsqueda documental;
- evitar que se redacte el plan sin base;
- detectar qué información existe;
- detectar qué información falta;
- organizar los temas específicos de un coworking/taller de carpintería artesanal;
- integrar la estructura general de Red ARCE con los módulos ampliados que requiere este proyecto.

Este documento no desarrolla el plan de empresa.  
Este documento no cierra decisiones.  
Este documento no sustituye la investigación pendiente.

## Nota de interpretación

Los subtemas de este documento no son necesariamente subtítulos del plan final.

Funcionan como temas de búsqueda, extracción y verificación documental.

Durante la redacción posterior, algunos podrán convertirse en apartados, otros se integrarán dentro de secciones mayores y otros quedarán solo como observaciones, vacíos o anexos.

---

## 2. Regla de extracción documental

Para cada tema o subtema, se deberá buscar información en las fuentes disponibles.

Si hay información suficiente:

- extraer la información relevante;
- indicar la fuente de procedencia;
- diferenciar entre dato, hipótesis, inferencia y decisión pendiente;
- señalar advertencias de uso;
- no convertir hipótesis en hechos.

Si no hay información suficiente:

- no inventar contenido;
- no completar con suposiciones;
- marcar el apartado como `VACÍO / PENDIENTE DE INVESTIGACIÓN`;
- explicar brevemente qué falta;
- dejar una observación para investigación posterior.

Formato mínimo para vacíos:

```text
VACÍO / PENDIENTE DE INVESTIGACIÓN:
No se encontró información suficiente sobre [tema].
Falta investigar/verificar: [dato, fuente, presupuesto, normativa, entrevista, validación, etc.].
```

---

## 3. Reglas de uso de fuentes

### 3.1. Fuentes primarias

Se deben priorizar para:

- precios;
- servicios;
- horarios;
- condiciones de acceso;
- cursos;
- membresías;
- datos oficiales;
- información legal o institucional;
- evidencia directa de mercado.

### 3.2. Fuentes secundarias

Se pueden usar para:

- análisis;
- interpretación;
- riesgos;
- comparaciones;
- hipótesis;
- aprendizajes;
- posibles estrategias.

No deben usarse como prueba definitiva sin verificación.

### 3.3. Fuentes históricas

Se pueden usar solo para:

- origen del proyecto;
- motivación inicial;
- propósito;
- valores;
- antecedentes.

No deben usarse para definir la estrategia actual si contienen enfoques antiguos, rurales o no vigentes.

### 3.4. Fuentes contaminadas

No se debe trasladar al plan contenido relacionado con:

- SISTREG;
- logística;
- software;
- agentes;
- automatizaciones;
- trazabilidad documental;
- cobro por evidencia;
- empresas de transporte;
- modelos ajenos al taller colaborativo.
